# Contributing

## Branching

Use short-lived branches from `main`:

- `feat/<topic>`
- `fix/<topic>`
- `docs/<topic>`
- `chore/<topic>`

## Commit examples

- `feat(api): add evaluation session endpoint`
- `fix(worker): preserve partial model results`
- `docs(architecture): record model registry decision`

## Pull requests

Every pull request should:

- explain the user or engineering outcome
- include or update tests
- pass linting, type checking, tests, and security scans
- document schema, prompt, rubric, or license changes
